# 基于 headless-tree 实现的树形组件需求说明（给AI Agent）

## 1. 整体技术栈
- 框架：**React**
- 核心库：**headless-tree**（lukasbach 版本）
- UI 风格：**完全无头，只提供逻辑，UI 100% 自定义渲染**

## 2. 节点类型与结构
- 节点分两类：
  - **folder（文件夹）**：可展开、可包含子节点、可接收拖拽
  - **file（文件）**：叶子节点，不可展开、不可放入子节点
- 数据结构：
  - 每个节点包含：`id`、`label`、`type: 'folder' | 'file'`
  - 支持**无限层级嵌套**

## 3. 异步懒加载（核心要求）
- **不一次性加载全量树**，所有节点通过**接口异步获取**
- 提供一个统一接口：
  - 接口：`/api/tree?parentId=xxx`
  - `parentId = null` → 获取**根节点**
  - `parentId = 文件夹ID` → 获取该节点的**直接子节点**
- 行为规则：
  1. 组件初始化 → 加载根节点
  2. 点击**展开 folder** → 自动调用接口加载其子节点
  3. 已加载过的节点**不再重复请求**
  4. 加载中显示 loading 状态
  5. 文件节点不触发加载

## 4. 基础交互
- 点击 folder 可**展开/收起**
- 节点根据层级自动**缩进**
- 支持**节点选中**（单选）
- 渲染必须使用 `visibleNodes`，只渲染可见节点

## 5. DND 拖拽能力
- 支持节点拖拽排序与移动
- 拖拽规则：
  - 只能拖入 **folder**
  - **file 不能作为拖入目标**
  - folder 可拖入 folder
  - file 可拖入 folder
  - file 不能拖入 file

## 6. 节点操作 API
需要实现并暴露以下操作：
- 新增节点（添加子节点）
- 删除节点（按 node / id 删除）
- 更新节点名称/属性
- 全选/取消全选
- 全部展开/全部收起
- 监听：选中变化、展开变化、拖拽完成、节点删除

## 7. UI 渲染要求
- 完全自定义 DOM 结构与样式
- folder 显示展开/收起箭头
- file 不显示箭头
- 支持 hover 态、选中高亮态
- 加载态展示 loading 图标
- 按设计稿可自由替换图标、颜色、间距、圆角

## 8. 交付要求
- 输出**完整可直接运行的 React 代码**
- 结构清晰，注释合理
- 接口部分用模拟函数，方便替换真实接口
- 不依赖额外 UI 库，可自由套样式
